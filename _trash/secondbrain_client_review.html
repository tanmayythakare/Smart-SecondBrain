Verdict: The visual design is genuinely premium — glassmorphism, dark theme, and typography are all strong. But there are 2 critical functional gaps (notes creation broken, sidebar data is fake), several major UX issues (user avatar shows "U", no priority/status on tasks, no empty state guidance), and many small polish issues that together make it feel unfinished. This is an 80% product pretending to be 100%.

Register screen ▶
minor
No password strength indicator
Password field accepts anything silently. No minimum length hint, no strength meter. User has no idea if "abc" is valid until they hit submit and potentially get a cryptic error.
minor
No password confirmation field
Standard registration has "confirm password." Without it, a typo means the user is locked out and doesn't know why.
minor
No "show password" toggle
Both register and login have no eye icon to reveal password. Basic UX expectation in 2026.
polish
Logo icon is generic and unlabeled
The purple icon above "Join Second Brain" looks like a balance/scale symbol. It doesn't relate to the product concept. A brain icon or custom logo mark would be far more on-brand.
polish
Large empty black background
The card floats in a completely black void. Even a subtle radial gradient, a pattern, or a faint grid would elevate the first impression significantly.
polish
Label spacing inconsistency
On register: "EMAIL" label is closer to the subtitle "Create your personal knowledge engine" than to its own input field. The visual grouping is wrong — labels should be tight to their field, not floating between sections.
Login screen ▶
major
No "Forgot password?" link
Every real app has this. Its absence will cause users to abandon if they forget credentials. Even if the feature doesn't exist yet, the link should be there as a placeholder.
minor
No error state shown
What happens when wrong credentials are entered? Is there an inline error message? From the screenshot there is no error feedback visible — this needs to be verified and shown.
polish
Login card is smaller than register card
The login card appears narrower and shorter than the register card. Inconsistent sizing between two closely related screens is noticeable and looks like an oversight.
Tasks screen ▶
major
No priority or status visible on task cards
Tasks "eat", "sleep", "Code" all show "Work · No due date" — no priority badge (Low/Med/High), no status chip (Todo/In Progress/Done). These fields exist in your DB but are invisible in the UI. The task list is essentially useless for prioritization.
major
Delete confirmation dialog covers the task content
In the delete state screenshot, the "Delete this task?" dialog sits inline inside the task card, covering the task title ("Code" is visible behind it). This is a confusing inline-overlay pattern. Should be a proper modal or a slide-in confirmation strip below the task.
major
Inline edit replaces title with a bare input field
When editing "Code", the title becomes a plain outlined input with no save/cancel buttons visible. User has no clear affordance to confirm or cancel the edit. Keyboard-only (Enter/Escape) is not discoverable.
minor
"3 active today" badge is misleading
Tasks "eat", "sleep", "Code" all have "No due date" — how are they "active today"? The badge either counts all incomplete tasks (not "today") or the logic is wrong. Either fix the label to "3 total" or fix the due date logic.
minor
Edit and delete icons inconsistent between states
In normal state: pencil icon (orange) + trash icon. In edit state: a floppy disk icon (purple/pink) + trash icon. Two different icons for "save" vs "edit" is fine, but the floppy disk looks out of place vs the clean pencil — consider a checkmark instead.
minor
No way to set due date or priority when adding a task
The "Add a new task..." input is a single plain text field. No date picker, no priority selector inline or expandable. Users can only set the title. Everything else requires... the AI chat? That's not obvious.
polish
Task category shows "Work" hardcoded for all tasks
Every task shows "Work" as the category. Either this is the default and can't be changed, or it's a hardcoded placeholder. If tasks can have categories, the UI should allow setting them.
polish
Checkbox has no visual feedback on hover
The task completion checkbox appears flat with no hover state. A subtle color change or scale on hover would make it feel more interactive.
Notes screen (Knowledge Base) ▶
critical
No way to create a note — the + button does nothing visible
The empty state says "Create your first note with the + button." There IS a floating + button (bottom right). But you confirmed it doesn't actually open a note creation form. This is a completely broken core feature. The Knowledge Base is a read-only page right now.
major
Search bar exists but there are no notes to search
The search field "Search your notes, ideas, and knowledge..." is prominently displayed above the "No notes yet" empty state. A search bar in an empty state is visually confusing and should either be hidden or shown only when notes exist.
minor
Empty state icon is a generic OS emoji
The "No notes yet" empty state uses what appears to be a system emoji (notepad with pen). This looks inconsistent with the premium glassmorphism design. Should be a custom SVG icon in the app's color scheme.
polish
Page title "Knowledge Base" doesn't match nav label "Notes"
The nav tab says "Notes" but the page heading says "Knowledge Base." Inconsistent naming creates confusion. Pick one and use it everywhere.
AI Chat screen ▶
critical
Sidebar data is static/fake — not the user's real data
"Upcoming" shows "Project sync with Sarah" and "Review feedback on PR" — these are not the user's real tasks ("eat", "sleep", "Code"). "Related Note" shows "Q2 Strategy Roadmap" — no notes exist yet. "Memory Insight" shows a hardcoded quote. All three sidebar panels are fake placeholder data, not live data from the DB.
major
User avatar shows "U" instead of username initial
Every user message bubble shows the letter "U" as the avatar. This is clearly a placeholder — it should show the first letter of the logged-in user's username or a proper avatar.
major
AI greets with "Good morning!" at 6:19 PM
The first AI message says "Good morning! I've analyzed your upcoming schedule." This was sent at 6:19 PM. The time-of-day greeting logic is wrong — or the AI is using a hardcoded greeting. Should use actual server time to determine morning/afternoon/evening.
major
AI claims to have "analyzed your upcoming schedule" but sidebar shows fake data
The greeting implies contextual awareness of the user's real schedule. But the upcoming tasks shown are not the user's real tasks. This is a false promise to the user — the AI is presenting fabricated context as if it's real personalization.
minor
Task items in AI response show no status, priority, or due date
When the AI lists tasks ("eat", "sleep"), only the title and "TASK" badge are shown. No due date, no priority, no status. These are clickable cards but they contain minimal information.
minor
"PRESS ENTER TO SEND" hint is unnecessary
The text below the input field is redundant — Enter to send is a universal standard. This screen real estate could be removed or used for a character counter or a shortcut hint that's actually non-obvious.
minor
"SECOND BRAIN REASONING LAYER" label is technical jargon
The small ALL-CAPS label above "AI Assistant" reads like an internal engineering term, not a user-facing label. A user doesn't need to know it's a "reasoning layer." Remove it or replace with something meaningful like "Powered by Gemini".
polish
Like/dislike buttons have no active/selected state
Thumbs up and thumbs down icons appear identical whether clicked or not. There's no visual confirmation that feedback was recorded. Should fill/highlight on selection.
polish
No suggested prompt chips on empty state
When the chat is empty (first load), there are no example prompts to guide new users. "Show my high priority tasks", "Summarize my notes", "What's due this week?" chips would dramatically improve first-time discoverability.
Global / navigation issues ▶
major
Settings icon (gear) does nothing
Top-right gear icon is present on every screen. If it opens a settings page, that wasn't shared. If it doesn't work yet — remove it. A non-functional UI element erodes trust immediately.
minor
No active page highlight consistency
The active nav item uses a filled pill. But the icon + label styling varies slightly between AI (star icon, white bg pill) vs Tasks (checkbox icon) vs Notes (document icon). The icon weights and sizes look inconsistent on close inspection.
minor
Floating + button (bottom right) has no tooltip
The purple sparkle/plus button bottom-right appears on Tasks and Notes screens. No tooltip on hover, no label. On the Notes screen it's the only way to create a note but the action is not working. Needs a tooltip and a working handler.
polish
No page transitions or loading states
Navigating between AI, Tasks, and Notes appears to be instant hard swaps with no transition animation. A 150ms fade would make the app feel more polished.
polish
"Logout" text in top-right is very prominent
The logout button with an arrow icon and full "Logout" text label sits in the primary navigation area. It's too prominent — most apps tuck logout into a profile dropdown. It makes the nav feel cluttered.
