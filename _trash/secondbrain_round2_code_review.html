app.component.html / app.component.ts 6 issues ▶
critical
User avatar hardcoded as "U" — never shows real username
In ai-chat.component.html: <div class="avatar user-avatar">U</div>. The AppComponent has AuthService injected but never reads the current user's name. Fix: inject AuthService into AiChatComponent and bind {{ currentUser?.username[0] | uppercase }}.
major
Theme defaults to dark but ignores OS preference
isDarkMode = true hardcoded, then overridden by localStorage. But if a user has never visited, it ignores prefers-color-scheme. Should check window.matchMedia('(prefers-color-scheme: dark)').matches as fallback before defaulting to dark.
major
Floating chat bubble navigates to /ai instead of opening inline
The .floating-chat-bubble has routerLink="/ai" — clicking it fully navigates away from the current page. A floating bubble should open an overlay panel without losing context. Either remove the bubble or implement it as a proper overlay.
minor
isLoginPage() checks for '/auth' but route might be '/auth/login' or '/auth/register'
router.url.includes('/auth') is fine as a contains-check, but if you ever add a route like /authenticated-dashboard, this breaks. Safer: router.url === '/auth/login' || router.url === '/auth/register'.
minor
lucide.createIcons() called via window cast — fragile
(window as any).lucide?.createIcons() in toggleDarkMode is a global side-effect call that relies on Lucide being loaded as a global script. If the script fails to load, icons silently break. Should use Angular's proper Lucide integration or a service.
polish
No loading/transition on route change
The <router-outlet> has no animation. Pages hard-swap instantly. Adding an Angular route animation (fade 150ms) takes ~10 lines and dramatically improves perceived quality.
ai-chat.component.html / ai-chat.component.css 8 issues ▶
critical
upcomingTasks, memoryInsight, relatedNote are hardcoded in the component — not from DB
The template binds *ngFor="let task of upcomingTasks", {{ memoryInsight }}, and {{ relatedNote.title }}. These must be populated from real API calls: tasks from TaskService.getTasks() filtered by upcoming due dates, and relatedNote from the last edited note via NoteService. Until wired, every user sees fake data.
critical
AI greeting time logic is missing — "Good morning" fires at any time
The initial greeting message sent to Gemini likely contains a hardcoded "Good morning" in the system prompt or first message in AiChatService.java. The backend prompt needs to use LocalTime.now() to dynamically generate "Good morning / afternoon / evening" based on server time.
major
Feedback buttons: only thumbs-up has active state — thumbs-down does not
Template: [class.active-feedback]="msg.hasFeedback" on thumbs-up button only. Thumbs-down has no equivalent. Also, after clicking either button, both remain visible — the condition *ngIf="!msg.hasFeedback" hides the whole section after feedback, which means the user can't see which they chose. Show selected state instead of hiding.
major
Suggested chips fire sendMessage() immediately on click — no confirmation
(click)="userInput = chip; sendMessage()" — sets input AND fires immediately. If a user accidentally taps a chip, they can't cancel. Better pattern: set userInput = chip and focus the textarea, let user press Enter to confirm.
major
textarea doesn't auto-grow — stays at rows="1"
The chat input is rows="1" with no auto-resize logic. If a user types a long message it scrolls inside a single-line box. Need a (input) handler that adjusts element.style.height dynamically, or use cdkTextareaAutosize from Angular CDK.
minor
Right sidebar "RELATED NOTE" hardcoded — never updates based on context
relatedNote.title and relatedNote.lastEdited are static. This panel should query the most recently edited note or the note most semantically related to the current conversation. At minimum, wire it to the last updated note from NoteService.
minor
No scroll-to-bottom on new message
The template has #scrollContainer ref but if the component doesn't call scrollContainer.scrollTop = scrollContainer.scrollHeight after each message, older conversations push new messages off-screen. Verify this is wired in the ts file.
polish
"PRESS ENTER TO SEND" hint is redundant and wastes space
Universal convention. Remove it or replace with a character counter or a useful hint like "Shift+Enter for new line" which is actually non-obvious.

major

Global CSS / design system issues 7 issues ▶
major
All colors hardcoded as hex — no CSS variables for theming
Every component uses raw hex: #161616, #1f1f1f, #6b7280, #6366f1 etc. scattered across every CSS file. When you want to change the brand color from indigo to something else, you'll need to find and replace across 8+ files. Create a global :root design token system: --color-surface, --color-border, --color-brand etc.
major
Light mode is partially broken — colors are hardcoded dark
The theme toggle exists and sets body.dark-theme, but component CSS files use hardcoded dark hex values (color: white, background: #161616) with no light-mode overrides. Switching to light mode likely shows white text on white background in many places.
minor
!important used excessively — 40+ instances across CSS files
Every input, button, and card override uses !important. This indicates specificity wars — likely a global CSS file is overriding component styles. Fix the root cause (scoped component styles or proper CSS specificity) instead of fighting it with !important.
minor
Emoji used as icons in action buttons throughout
✏️ 🗑️ 📋 💡 ✅ ✨ used as UI icons. Emoji render differently across Windows/Mac/Android — sizes, colors, and styles vary significantly. The app uses SVG icons in the nav but emoji in components. Standardize to SVG everywhere.
minor
No focus-visible styles anywhere
Keyboard navigation is invisible. No :focus-visible outline on buttons, inputs, or links. This fails basic accessibility requirements and makes the app unusable without a mouse.
polish
Font sizes use rem with non-standard values (0.65rem, 0.72rem, 0.78rem)
The app has font sizes like 0.65rem, 0.72rem, 0.78rem, 0.825rem, 0.875rem, 0.9rem, 0.95rem — 7+ different sizes with no clear type scale. Define a scale: xs=0.75rem, sm=0.875rem, base=1rem, lg=1.125rem and stick to it.
polish
box-shadow used in multiple places despite glassmorphism aesthetic
Several hover states add box-shadow: 0 4px 12px rgba(100,108,255,0.2) and 0 8px 24px rgba(0,0,0,0.3). These glow/shadow effects conflict with the flat dark glassmorphism design. Pick one aesthetic and be consistent.
